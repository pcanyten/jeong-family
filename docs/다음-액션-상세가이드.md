# SQL 성공 후 — 다음 액션 상세 가이드

SQL 실행까지 완료하셨다면, 아래 순서대로 진행하세요.

---

## STEP 1. Supabase에서 “연결 정보” 복사하기

우리 웹이 Supabase와 통신하려면 **주소(URL)** 와 **비밀 키(anon key)** 가 필요합니다.

1. Supabase 대시보드 **왼쪽 메뉴**에서 **톱니바퀴 아이콘** 클릭 → **Project Settings** 로 들어갑니다.
2. 왼쪽에서 **API** 메뉴를 클릭합니다.
3. 화면에서 아래 두 개를 **메모장이나 메모 앱에 복사**해 두세요.
   - **Project URL**  
     예: `https://abcdefghijk.supabase.co`
   - **Project API keys** 안에 있는 **anon public** (긴 문자열 하나)  
     예: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...` (매우 긴 글자)

나중에 Vercel 배포할 때 똑같이 넣을 거라서, 잃어버리지 않게 저장해 두세요.

---

## STEP 2. Supabase Storage 버킷 2개 만들기

피드 사진과 사진첩 사진을 저장할 공간을 만듭니다.

1. Supabase 왼쪽 메뉴에서 **Storage** 클릭.
2. **New bucket** 버튼 클릭.
3. **첫 번째 버킷**
   - Name: `feed` (소문자, 그대로 입력)
   - **Public bucket** 에 체크
   - **Create bucket** 클릭
4. 다시 **New bucket** 클릭.
5. **두 번째 버킷**
   - Name: `gallery` (소문자, 그대로 입력)
   - **Public bucket** 에 체크
   - **Create bucket** 클릭

이제 Storage 목록에 `feed`, `gallery` 두 개가 보이면 완료입니다.

---

## STEP 3. (선택) 로컬에서 먼저 실행해 보기

배포 전에, 내 컴퓨터에서 화면이 잘 뜨는지 확인하려면:

1. **Node.js 설치**  
   - [https://nodejs.org](https://nodejs.org) 에서 LTS 버전 다운로드 후 설치.
2. **터미널(또는 명령 프롬프트)** 열기.
3. 프로젝트 폴더로 이동 후 패키지 설치 및 실행:
   ```bash
   cd "정씨네 이야기 프로젝트가 있는 폴더/jeong-family"
   npm install
   npm run dev
   ```
4. 브라우저에서 **http://localhost:3000** 접속.
5. **환경 변수 설정**  
   - 프로젝트 폴더에 `.env.local` 파일이 없으면 새로 만듭니다.
   - 아래 두 줄을 넣고, STEP 1에서 복사한 값으로 바꿉니다.
     ```
     NEXT_PUBLIC_SUPABASE_URL=여기에_Project_URL_붙여넣기
     NEXT_PUBLIC_SUPABASE_ANON_KEY=여기에_anon_public_키_붙여넣기
     ```
   - 저장한 뒤 `npm run dev` 를 다시 실행하고, 다시 http://localhost:3000 접속.

여기서 로그인·가입 화면이 보이면 준비가 된 것입니다.  
(로컬은 건너뛰고 바로 배포해도 됩니다.)

## STEP 4. Vercel에 배포하기 (실제 서버에 올리기)

### 4-1. Vercel 가입 및 프로젝트 가져오기

1. [https://vercel.com](https://vercel.com) 접속.
2. **Sign Up** → GitHub 또는 이메일로 가입.
3. 로그인 후 **Add New…** → **Project** 클릭.
4. **Import Git Repository** 에서:
   - GitHub에 이미 `jeong-family` 코드를 올려두었다면: 해당 저장소 선택 후 **Import**.
   - GitHub에 없다면:
     - [https://github.com](https://github.com) 가입 후, **New repository** 로 `jeong-family` 저장소 생성.
     - 컴퓨터의 `jeong-family` 폴더를 그 저장소에 올린 뒤(푸시), Vercel에서 그 저장소를 Import.

### 4-2. 환경 변수 넣기 (매우 중요)

1. 저장소 Import 후 **Configure Project** 화면이 나옵니다.
2. **Environment Variables** 라는 칸을 찾습니다.
3. 아래처럼 **이름은 정확히** 넣고, 값은 STEP 1에서 복사한 것으로 채웁니다.

   | Name (이름) | Value (값) |
   |------------|------------|
   | `NEXT_PUBLIC_SUPABASE_URL` | STEP 1에서 복사한 **Project URL** 전체 |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | STEP 1에서 복사한 **anon public** 키 전체 |

4. 각각 **Key** 와 **Value** 입력 후 **Add** 또는 체크 후 **Deploy** 진행.

### 4-3. 배포 완료 후

1. **Deploy** 가 끝나면 **Visit** 또는 `https://프로젝트이름.vercel.app` 같은 주소가 나옵니다.
2. 그 주소로 접속해서 **로그인/가입 화면**이 보이면 배포 성공입니다.

## STEP 5. 첫 “관리자” 계정 만들기

가입한 사람 중 한 명을 관리자로 지정해야, 그 사람이 다른 가족을 “승인”할 수 있습니다.

1. 방금 배포한 사이트 주소에서 **가입**을 한 번 합니다. (이메일, 비밀번호, 이름 입력.)
2. Supabase 대시보드로 돌아갑니다.
3. 왼쪽 메뉴 **Table Editor** 클릭.
4. **profiles** 테이블을 클릭합니다.
5. 목록에서 **방금 가입한 이메일**에 해당하는 행을 찾습니다.  
   (몇 개 컬럼만 보여질 수 있음. 옆으로 스크롤 가능.)
6. 그 행에서:
   - **is_approved** → `false` 를 **true** 로 변경.
   - **is_admin** → `false` 를 **true** 로 변경.
7. **Save** 또는 저장 버튼 클릭.

이제 그 계정으로 사이트에 로그인하면:

- “승인 대기”가 아니라 **메인 피드**로 들어가고,
- 하단에 **회원 관리** 메뉴가 보이며,
- 다른 가족이 가입하면 **회원 관리**에서 **승인** 버튼으로 허용할 수 있습니다.

---

## STEP 6. 이후 사용 흐름

1. **관리자**: 로그인 → 피드/일정/사진첩 사용 + **회원 관리**에서 가입한 가족 **승인**.
2. **가족**: 가입 → “승인 대기” 화면에서 대기 → 관리자가 승인하면 로그인 후 메인 사용 가능.
3. **모든 승인된 가족**: 피드에 사진/글 올리기, 일정 등록, 사진첩 앨범·사진 추가.
