# 정씨네 이야기 — 실제 서버에 올리기 (초보자용)

이 문서는 개발자가 아니어도 따라할 수 있도록, **어떤 준비가 필요한지**와 **대략 어떤 순서로 하면 되는지**만 쉽게 설명합니다.

---

## 1. 전체적으로 필요한 것 (한눈에 보기)

| 준비물 | 뭐하는 곳인지 | 비용 |
|--------|----------------|------|
| **Supabase** | 회원/로그인, DB, 사진 저장 | 무료 플랜 있음 |
| **Vercel** | 우리 웹사이트를 24시간 켜두는 “서버” | 무료 플랜 있음 |
| **이메일 주소** | Supabase, Vercel 가입용 | - |
| **GitHub 계정** (선택) | 코드 올려두고 Vercel이 자동 배포하게 할 때 | 무료 |

**정리:** Supabase에서 “데이터·회원·사진”을 관리하고, Vercel에서 “웹 화면”을 서버에 올려 둔다고 생각하면 됩니다.

---

## 2. Supabase 준비 (데이터·회원·사진)

### 2-1. Supabase가 하는 일

- **회원 가입/로그인** (이메일·비밀번호)
- **데이터 저장** (게시글, 일정, 앨범, 댓글 등)
- **사진 파일 저장** (피드 사진, 사진첩)

우리 앱은 전부 Supabase에 연결해서 쓰도록 만들어져 있습니다.

### 2-2. 해야 할 일 (체크리스트)

1. **가입**
   - [supabase.com](https://supabase.com) 접속
   - “Start your project” → Google/이메일로 가입

2. **새 프로젝트 만들기**
   - “New Project” 클릭
   - 이름: 예) `jeong-family`
   - 비밀번호 한 번 정하고, 리전은 가까운 것(예: Northeast Asia) 선택
   - “Create new project” → 생성될 때까지 1~2분 기다리기

3. **연결 정보 복사해 두기**
   - 왼쪽 메뉴 **Project Settings**(톱니바퀴) 클릭
   - **API** 메뉴 들어가기
   - 아래 두 개를 메모장에 복사해 두기:
     - **Project URL** (예: `https://xxxx.supabase.co`)
     - **anon public** 키 (긴 문자열 하나)

4. **데이터베이스 테이블 만들기**
   - 왼쪽 메뉴 **SQL Editor** 클릭
   - 우리 프로젝트 안에 있는 `supabase/schema.sql` 파일을 연 다음, **전체 내용을 복사**
   - SQL Editor에 붙여넣고 **Run** 버튼 클릭
   - “Success” 나오면 테이블 생성 완료

5. **사진 저장용 버킷 만들기**
   - 왼쪽 메뉴 **Storage** 클릭
   - “New bucket” 클릭
   - 이름: `feed` → Public 체크 → Create
   - 다시 “New bucket” → 이름: `gallery` → Public 체크 → Create

6. **첫 관리자 정하기 (나중에 한 번만)**
   - 먼저 우리 웹에서 **가입**을 한 번 합니다 (실제 서버 주소에서).
   - Supabase 왼쪽 메뉴 **Table Editor** → **profiles** 테이블 클릭
   - 방금 가입한 이메일의 **한 줄** 찾기
   - 그 줄에서 **is_approved**, **is_admin** 값을 **true**로 바꾸고 저장  
   → 이 계정이 “관리자”가 되고, 다른 가족을 승인할 수 있습니다.

## 3. Vercel 준비 (웹사이트를 서버에 올리기)

### 3-1. Vercel이 하는 일

- 우리가 만든 “정씨네 이야기” 웹 화면을 **항상 켜 둔 서버**에 올려 줍니다.
- 주소가 생깁니다. 예: `jeong-family.vercel.app`
- 누구나 그 주소로 들어가면 로그인·피드·일정·사진첩을 쓸 수 있습니다.

### 3-2. 해야 할 일 (체크리스트)

1. **가입**
   - [vercel.com](https://vercel.com) 접속
   - “Sign Up” → GitHub 또는 이메일로 가입

2. **우리 프로젝트 올리기 (두 가지 중 하나)**

   **방법 A — 컴퓨터에서 직접 올리기**
   - Vercel 웹에서 “Add New” → “Project”
   - “Import Third-Party Git Repository” 말고, **로컬 폴더를 끌어다 놓거나**  
     “Import”에서 **Upload**로 우리 `jeong-family` 폴더를 올립니다.
   - (로컬 업로드는 제한이 있을 수 있어서, 아래 방법 B를 추천합니다.)

   **방법 B — GitHub 써서 자동 배포 (추천)**
   - [github.com](https://github.com) 가입 후, 새 저장소(Repository) 하나 만듦. 이름 예: `jeong-family`
   - 컴퓨터에 “정씨네 이야기” 프로젝트 폴더가 있다면,  
     GitHub 데스크톱이나 웹에서 이 폴더를 그 저장소에 “푸시”합니다.
   - Vercel에서 “Add New” → “Project” → “Import Git Repository”에서 방금 만든 `jeong-family` 저장소 선택
   - “Import” 클릭

3. **환경 변수 넣기 (매우 중요)**
   - Vercel이 우리 프로젝트를 “설정”하는 단계에서 **Environment Variables**가 나옵니다.
   - 여기서 아래 두 개를 **이름 그대로** 추가합니다.

   | Name | Value |
   |------|--------|
   | `NEXT_PUBLIC_SUPABASE_URL` | Supabase에서 복사한 **Project URL** |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase에서 복사한 **anon public** 키 |

   - 각각 “Add” 한 다음 **Deploy** 진행

4. **배포 끝나면**
   - “Visit” 또는 생성된 주소(예: `https://jeong-family.vercel.app`)로 들어가 봅니다.
   - 로그인 화면이 보이면 성공입니다.  
   - Supabase에서 첫 가입 계정의 `is_approved`, `is_admin`을 true로 바꿔 두면, 그 계정으로 로그인해서 피드·일정·사진첩·회원 승인을 쓸 수 있습니다.

---

## 4. 준비 순서 요약 (실제로 할 때 순서)

1. **Supabase** 가입 → 프로젝트 생성 → URL·anon 키 복사  
2. **SQL Editor**에서 `schema.sql` 전체 실행 → 테이블 생성  
3. **Storage**에서 `feed`, `gallery` 버킷 생성  
4. **Vercel** 가입 → 우리 프로젝트(GitHub 또는 업로드) 연결  
5. Vercel **환경 변수**에 Supabase URL, anon 키 입력  
6. **Deploy** 후 나온 주소로 접속  
7. 웹에서 **한 번 가입** → Supabase **profiles**에서 그 계정을 `is_approved`, `is_admin` true로 변경  
8. 그 계정으로 로그인해서 사용

## 5. 자주 하는 질문 (초보자용)

**Q. “서버”가 뭔가요?**  
A. 웹사이트가 24시간 돌아가게 해 주는 컴퓨터라고 생각하면 됩니다. Vercel이 그 역할을 해 줍니다.

**Q. Supabase랑 Vercel 둘 다 꼭 써야 하나요?**  
A. 네. 우리 앱은 “화면”은 Vercel에, “회원·데이터·사진”은 Supabase에 연결되어 있어서 둘 다 필요합니다.

**Q. 비용은 얼마나 나오나요?**  
A. 가족만 쓰는 수준이면 Supabase·Vercel 무료 플랜으로 충분한 경우가 많습니다. 사용량이 커지면 각 서비스 요금제를 확인하면 됩니다.

**Q. 주소를 “정씨네이야기.com”처럼 하고 싶어요.**  
A. 도메인을 구매한 뒤, Vercel 프로젝트 설정에서 “Domains”에 그 도메인을 연결하면 됩니다. (도메인 구매는 다른 사이트에서 합니다.)

**Q. 코드를 수정했는데 반영이 안 돼요.**  
A. GitHub에 푸시해 두었다면 Vercel이 자동으로 다시 배포합니다. 몇 분 기다리거나 Vercel 대시보드에서 “Redeploy”를 눌러 보세요.
