# 승인 대기 화면에서 안 넘어갈 때 — Service Role 키 추가

프로필에 `is_approved = true` 인데도 "승인 여부 확인"을 눌러도 메인으로 안 넘어가면, **Service Role 키**를 넣어 주세요. 서버가 RLS 없이 프로필을 읽을 수 있게 됩니다.

## 1. Supabase에서 키 복사

1. Supabase 대시보드 → 왼쪽 아래 **Project Settings** (톱니바퀴)
2. **API** 메뉴 클릭
3. **Project API keys** 에서 **service_role** (Secret 키) 찾기
4. **Reveal** 또는 복사 버튼으로 **전체 키** 복사  
   (절대 클라이언트/브라우저에 노출하지 말 것)

## 2. .env.local 에 추가

프로젝트 폴더(jeong-family) 안의 **.env.local** 파일을 열고, 아래 한 줄을 **추가**합니다.

```
SUPABASE_SERVICE_ROLE_KEY=여기에_방금_복사한_service_role_키_붙여넣기
```

- 등호 앞뒤 공백 없이
- 따옴표 없이 값만

저장 후 **npm run dev** 를 한 번 중지했다가 다시 실행하세요.

## 3. 다시 테스트

1. 브라우저에서 **http://localhost:3000/waiting** 접속
2. **승인 여부 확인** 클릭
3. 메인(피드) 화면으로 넘어가면 성공입니다.

이 키는 **서버에서만** 사용되며, 브라우저로 노출되지 않습니다.
