# 승인했는데도 "승인 대기중" 화면이 나올 때

## 1. 테이블 구분하기

Supabase에는 비슷한 데이터가 **서로 다른 테이블**에 있습니다.

| 테이블 | 위치 | 칸(컬럼) 예시 | 용도 |
|--------|------|----------------|------|
| **auth.users** | Authentication 쪽 / DB의 auth 스키마 | id, email, aud, role | 로그인 계정 |
| **public.profiles** | Table Editor → **public** → **profiles** | id, display_name, **is_approved**, **is_admin**, created_at | 앱에서 쓰는 "승인" 여부 |

- **승인(is_approved)** 을 바꿔야 하는 곳은 **public.profiles** 입니다.
- 스크린샷에 **aud, role, email** 만 보이면 그건 **auth.users** 이고, **is_approved / is_admin** 이 보여야 **profiles** 입니다.

## 2. 꼭 확인할 것

1. 왼쪽에서 **Table Editor** 클릭
2. **스키마가 public** 인지 확인 (auth 아님)
3. 테이블 이름 **profiles** 클릭
4. 위에 칸 이름에 **is_approved**, **is_admin** 이 있는지 확인
5. **id = 8f51c8fd-b7cc-4a16-849d-0f1db597be28** 인 **한 행**에서  
   **is_approved = true**, **is_admin = true** 로 되어 있는지 확인

## 3. 그래도 안 되면

- 브라우저 **캐시 삭제** 또는 **시크릿(프라이빗) 창**에서 로그인
- 터미널에서 **npm run dev** 중지 후 다시 실행
- 다시 로그인 후 **승인 여부 확인** 버튼 클릭
